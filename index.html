Privacy Policy

Effective: June 11, 2026
Last updated: June 11, 2026

1. Introduction

Thanks for using kuyo. Before you add it to your server, read this so you know how your data is handled.

Your privacy matters to me. I only collect what the bot needs to work. I don't sell your data and I don't share it with third parties.

If something here looks wrong, email business@mutuals.cc.

2. Summary

kuyo uses Discord user IDs, server IDs, channel IDs, role IDs, and your server's saved settings to run commands and features.

kuyo reads message content when a feature needs it, like prefix commands, autoresponders, counting, sticky messages, and moderation tools. That content is only used to run the feature. kuyo does not keep a log of your server's chat.

If welcome messages, invite tracking, or role based features are enabled, kuyo uses member join, leave, and role change events to make those work.

3. Using kuyo in Discord

When you add kuyo to your server, you allow it to see and process information it has access to on Discord, only to run the features you have set up.

3.1. Commands and message features

kuyo reads message content when you run a prefix command or when a configured feature needs to check a message. Slash commands do not need to read your messages. Message content is processed to run the feature and is not kept as a chat log.

3.2. Welcome, invites, and member events

If you enable welcome messages or invite tracking, kuyo stores the member and invite data those features need. Role based automation uses role change events the same way.

3.3. Server settings

Settings you configure through kuyo (prefixes, channels, templates, feature toggles) are stored so the bot remembers them between restarts.

4. What we don't collect

kuyo does not collect Discord passwords, login tokens, or payment and financial information.

5. How your data is shared

I do not sell your data. I do not share it with advertisers, data brokers, or other third parties.

6. Security

kuyo's database and servers are not open to the public. Your server's configuration and feature data is only accessible through systems I operate for running the bot.

7. How long data stays

Your server's data stays stored while kuyo is in that server.

When you remove kuyo, that server stops collecting new data. Stored data for that server is automatically deleted after six months.

You can request earlier deletion at any time by emailing business@mutuals.cc with your Discord user ID.

8. Your rights

You can:

  • Request what data I hold about you or your server
  • Request deletion of your data at any time
  • Remove kuyo from your server to stop future data collection

Email business@mutuals.cc for access or deletion requests. Include your Discord user ID, not just your username. I will respond to access requests within seven days and deletion requests within fourteen days.

9. Children's privacy

kuyo does not knowingly collect data from users under 13. Discord requires users to be at least 13 years old.

10. Changes

I may update this page from time to time. If you keep using kuyo after an update, that means you accept the new version.

11. Contact

Questions about this policy or your data: business@mutuals.cc
